## M7.1 文档权限

状态：功能实现完成，完整发布验收待处理 M6 基线 E2E 失败。

已完成：

- 文档直接路由 `/documents/:documentId`，未认证返回登录流程，无权访问返回统一 404。
- 私有/团队文档策略、显式 `editor`/`viewer` 权限和 owner 管理权限。
- `created_by` 历史回填固定使用工作区最早加入的 owner（按 `created_at`、`user_id` 排序）；该字段只影响作者元数据，不削弱任何 owner 的管理权限。
- 文档读写、历史读取/恢复、文件附件和协作 WebSocket 统一使用文档授权判定。
- 工作区目录过滤不可访问私有文档；远程模式只保存活动文档。
- 远程文档新建、复制、删除 API；删除后服务端为当前用户选择仍可访问的活动文档。

验证记录：

- `pnpm test --run`：120 个测试文件、745 项通过。
- 真实 PostgreSQL：4 个文件、7 项通过；并发迁移测试使用单线程执行以避免临时 schema 初始化超出 hook 超时。
- M7 目标 E2E：`document-permissions.spec.ts`、`workspaces.spec.ts` 共 3 项通过。
- `pnpm build`：通过；Docker Web 镜像构建和服务健康检查通过。

完整 E2E 当前仍有 M6 既有成员导航、协作输入时序和附件上传失败，不能标记为完整发布验收通过。
