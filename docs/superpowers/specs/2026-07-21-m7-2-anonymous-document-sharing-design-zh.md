# M7.2 匿名文档分享设计

更新日期：2026-07-21

## 1. 目标

M7.2 在 M7.1 文档级授权边界之上增加可撤销的匿名只读分享。工作区所有者可以为文档生成高熵分享链接、设置过期时间、重新生成或关闭链接；未登录访问者只能读取经过清洗的最新持久化文档快照和受控附件。

本阶段不提供匿名编辑、评论、历史版本、任务分配、协作 WebSocket、分享密码、下载限制、域名白名单或多个并行分享链接。

## 2. 已确认的产品规则

- 每篇文档最多存在一个未撤销的分享链接。
- 新链接默认 24 小时后过期。
- 所有者可以选择 1 小时、24 小时、7 天、30 天或自定义过期时间。
- 自定义时间必须晚于当前时间，最长不超过 365 天。
- 创建与重新生成时只返回一次完整分享 URL；刷新后服务端只返回状态和过期时间。
- 重新生成会在同一事务内撤销旧链接并创建新链接。
- 关闭链接、重新生成或到期后，旧页面请求和已签发附件 URL 均失效。
- 只有工作区 owner 可以管理匿名分享；`created_by` 仍只表示作者元数据，不授予分享管理权限。

## 3. 数据模型

新增 `document_share_links`：

| 字段 | 约束 | 说明 |
| --- | --- | --- |
| `id` | 主键 | 不透明分享记录 ID |
| `workspace_id` | 非空 | 与 `document_id` 组成文档复合外键 |
| `document_id` | 非空 | 引用 `editor_documents(workspace_id, id)`，文档删除时级联删除 |
| `token_hash` | 非空、唯一 | 原始令牌的域隔离 HMAC |
| `created_by` | 非空、外键 | 创建或重新生成链接的 owner |
| `expires_at` | 非空 | 绝对过期时间戳 |
| `revoked_at` | 可空 | 关闭、重新生成或权限模式切换的撤销时间 |
| `created_at` | 非空 | 创建时间 |
| `updated_at` | 非空 | 最近生命周期更新时间 |

部分唯一索引约束同一 `(workspace_id, document_id)` 最多一条 `revoked_at IS NULL` 的记录。创建或重新生成时先撤销现有未撤销记录，再插入新记录。已过期记录保留用于审计和识别旧令牌，但创建新链接前同样会被标记撤销。

迁移必须幂等，不为现有文档自动创建公开链接，不迁移旧前端模拟分享状态，也不保存任何原始令牌。

## 4. 令牌与签名

- 原始分享令牌由 Node.js `randomBytes(32)` 生成，并使用 Base64URL 编码，提供 256 bit 随机性。
- 数据库查找键使用现有服务端哈希密钥做域隔离 HMAC-SHA-256，例如 `document-share:v1` 命名空间。
- 原始令牌只存在于创建响应、用户复制的 URL 和访问请求中；不得进入数据库、审计元数据或应用日志。
- 比较使用固定长度摘要和恒定时间语义。
- 分享响应设置 `Cache-Control: no-store` 与 `Referrer-Policy: no-referrer`。

## 5. 管理 API

路由：`/api/documents/:documentId/share-links`

### `GET`

要求已登录且对文档具有 `manage` 权限。返回：

```json
{
  "shareLink": {
    "id": "share-...",
    "status": "active",
    "expiresAt": 1784707200000
  }
}
```

不存在链接时 `shareLink` 为 `null`；已过期但未撤销时返回 `expired`。响应永不返回原始令牌或可复制 URL。

### `POST`

请求体为 `{ "expiresAt": number }`。服务端验证时间晚于当前时间且不超过 365 天；未提供时使用当前时间加 24 小时。操作在事务中撤销旧链接、创建新链接并写审计。

仅本次响应返回：

```json
{
  "shareLink": {
    "id": "share-...",
    "status": "active",
    "expiresAt": 1784707200000,
    "url": "https://example.test/share/<raw-token>"
  }
}
```

首次创建和重新生成使用同一端点与事务语义。

### `DELETE`

撤销当前未撤销链接并写审计。重复关闭保持幂等，返回 `204`。从文档策略的 `link` 模式切换到 `private` 或 `workspace` 时，也必须在替换策略的同一事务内撤销当前链接。

管理端点沿用 M7.1 的统一文档管理授权：未认证返回 `401`，没有管理权限或文档不存在统一返回 `404`，输入时间无效返回 `400`。

## 6. 匿名文档读取

新增：

- 页面路由：`/share/:token`
- 数据路由：`GET /api/shared-documents/:token`

匿名读取不创建登录会话，不接受写入，不加入 Yjs room，只读取最新持久化快照。服务端按令牌 HMAC 查找分享记录，并依次检查：

1. 分享记录存在。
2. `revoked_at` 为空。
3. `expires_at` 晚于当前时间。
4. 文档与工作区仍存在且工作区未删除。

未知令牌返回 `404`。可以关联到分享记录但已关闭、已重新生成或已过期的令牌返回 `410`。分享页面为这些状态提供简洁的不可用页面，不泄露文档标题或工作区信息。

## 7. 公开快照清洗

匿名响应只包含：

- 文档标题。
- 可展示的块类型、层级、正文和公开渲染所需数据。
- 经验证并替换后的图片或文件访问 URL。
- 分享链接失效时间。

清洗器必须：

- 清空块评论。
- 清空负责人、任务状态和到期日。
- 不返回成员、邮箱、作者身份、权限列表、历史版本、审计记录或内部工作区信息。
- 对 `image` 与 `file` 块校验 `data.key` 必须存在于 `document_attachments`，且属于当前工作区和文档；校验失败时移除附件数据。
- 保留普通链接卡片的公开 URL，但不把内部 `/api/files/` 地址原样返回。

清洗逻辑作为独立纯函数实现并进行单元测试。分享页复用现有只读块渲染能力，但不挂载工作区侧栏、编辑控件、评论、成员、历史和协作状态。

## 8. 附件签名代理

公开快照中的附件 URL 指向应用内签名代理，不直接暴露对象存储地址或裸分享令牌。签名载荷覆盖：

- 分享记录 ID。
- 附件对象 key。
- URL 失效时间。
- 签名版本。

附件 URL 默认 5 分钟失效。代理路由每次请求均验证签名、时间、附件归属，并再次读取 `document_share_links` 检查撤销和分享过期状态。由此保证即使附件 URL 尚未到达自身过期时间，关闭、重新生成或分享过期后也会立即拒绝访问。

附件响应使用只读 `GET`、私有缓存策略和原始 MIME 类型；不存在、归属不符、签名错误或分享失效统一拒绝，不提供上传、覆盖或目录读取能力。

## 9. 分享面板

`SharePopover` 在“拥有链接的人可查看”区域显示：

- 过期预设：1 小时、24 小时、7 天、30 天、自定义。
- 默认选中的 24 小时。
- 创建、复制、重新生成和关闭操作。
- 当前状态和准确过期时间。

完整 URL 只保存在创建或重新生成后的前端内存状态中。刷新后只能看到链接状态与过期时间，复制按钮不可用，所有者需要重新生成才能获得新 URL。操作期间按钮进入禁用态；错误通过面板内状态提示，不覆盖其他文档权限设置。

分享页使用独立、响应式的只读文档布局，桌面与移动端均不得出现工作区管理或编辑入口。

## 10. 审计与隐私

复用 `workspace_audit_events`，记录：

- `document_share.created`
- `document_share.regenerated`
- `document_share.revoked`
- `document_share.accessed`
- `document_share.access_denied`
- `document_share_attachment.accessed`
- `document_share_attachment.access_denied`

管理事件的 actor 为操作 owner；匿名访问事件 actor 为空。元数据只包含分享 ID、文档 ID、结果、拒绝原因和相关时间，不包含正文、评论、对象内容、原始令牌或完整 URL。

完全未知的随机令牌无法关联工作区，因此只返回 `404` 并计入聚合安全指标，不写 `workspace_audit_events`。已过期或已撤销的已知令牌访问写入对应工作区审计。

## 11. 测试与验收

### 单元测试

- 令牌长度、Base64URL 格式、HMAC 稳定性与域隔离。
- 默认 24 小时、全部预设、自定义时间上下界。
- 公开快照清洗评论、负责人、状态、内部字段与非法附件。
- 附件签名篡改、超时和固定时间比较。

### PostgreSQL 集成测试

- 迁移幂等及复合外键。
- 单文档单未撤销链接约束。
- 创建、重新生成、关闭和过期判断。
- 重新生成后旧 token hash 仍可识别为已撤销，但不能读取文档。
- 文档删除级联清理分享记录。
- 审计元数据不含令牌、URL 或正文。

### API 与组件测试

- 仅 owner 可管理，editor、viewer、非成员统一不能管理。
- GET 不泄露 URL，POST 只返回一次 URL，DELETE 幂等。
- 匿名响应字段白名单和 `404`/`410` 语义。
- 切换离开 `link` 模式自动撤销。
- 分享面板的预设、自定义时间、禁用态和生命周期操作。

### Playwright

- owner 创建默认 24 小时链接，未登录上下文可打开只读页面。
- 自定义过期时间正确显示。
- 匿名页面没有写入、历史、评论、成员或协作入口。
- 重新生成后旧链接立即返回失效，新链接可访问。
- 关闭和过期后页面与已签发附件 URL 均失效。
- 桌面和移动视口无布局遮挡。

## 12. 发布与回滚

发布前运行幂等迁移、全量单元测试、真实 PostgreSQL 集成、M7.2 Playwright 和生产构建。新表在未创建链接时不改变现有 M7.1 权限行为。

应用回滚时保留 `document_share_links` 与审计数据，不删除历史记录；旧应用不读取该表，匿名路由随应用回滚不可用。重新发布后仍按数据库中的撤销和过期状态执行，不恢复已失效链接。
