# Nexus Logo 与页签图标设计

## 背景

Nexus 当前没有浏览器页签图标。应用内使用深色圆角方块和白色字母 `N` 作为品牌标识，但该标识分别以文本节点散落在侧栏、登录页、加载页和错误页，无法作为统一资源复用。

## 目标

- 将现有黑底白色 `N` 标识固化为项目 Logo。
- 浏览器页签与所有应用内品牌位置使用同一 SVG 资源。
- 保持现有 32px、36px 和 40px 显示尺寸及页面布局不变。
- 在 16px favicon 尺寸下保持轮廓清晰。

## 视觉设计

Logo 使用 `32 × 32` 画布、深黑色 `#18181b` 圆角方形背景和近白色 `#fafafa` 几何字母 `N`。背景圆角为 6px，对应现有 `rounded-md` 的紧凑观感。字母使用填充路径而不是字体文本，避免不同浏览器和操作系统的字体渲染差异。

不增加渐变、阴影、描边或额外装饰。应用内现有容器阴影可由组件类名保留，图标资源本身保持简洁。

## 资源与组件

- 新增 `public/nexus-logo.svg`，作为唯一 Logo 图形资源。
- 新增共享 `BrandMark` React 组件，渲染该 SVG，并接受尺寸和阴影等展示类名。
- 侧栏、登录页桌面品牌区、登录页移动品牌区、加载页和错误页统一使用 `BrandMark`。
- `src/app/layout.tsx` 的 metadata 显式将 `/nexus-logo.svg` 配置为页签图标。

页面内 Logo 为装饰图形，使用空替代文本和 `aria-hidden`，因为相邻位置已有 `Nexus` 文本或明确的状态标签。页签图标继续由页面标题 `Nexus` 提供可识别名称。

## 测试与验证

- 组件测试验证 `BrandMark` 引用唯一 SVG 资源并保持装饰语义。
- metadata 测试验证页签图标指向 `/nexus-logo.svg`。
- 现有登录和编辑器组件测试验证替换后页面行为不变。
- 静态扫描确认 5 处手写字母 Logo 已全部替换。
- 使用浏览器检查桌面和移动登录页、工作区侧栏及页签请求，确认图标非空、清晰且不改变布局。
- 运行完整 Vitest 测试和 TypeScript 检查。

## 非目标

- 不修改 Nexus 名称、品牌文案、页面配色或布局。
- 不增加动画、主题变体、Apple Touch Icon 或安装应用图标。
- 不生成 PNG 或 ICO 副本。
