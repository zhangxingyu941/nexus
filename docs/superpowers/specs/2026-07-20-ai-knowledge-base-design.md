# AI 文字知识库设计

**日期：**2026-07-20

## 范围

本设计为 Nexus 增加纯文字 AI 知识库：索引应用已持久化的文档与区块，检索相关工作区内容，再把检索上下文发送给用户选择的 OpenAI 兼容聊天 API。

本期明确不包含语音输入和输出。仅存在于 IndexedDB 的本地工作区也不包含在内，因为服务端无法为它们构建和维护共享索引。

## 已确认的产品决策

- 向量保存在现有 PostgreSQL 数据库中，使用 `pgvector`。
- 每位用户自行配置 OpenAI 兼容的 `Base URL`、API Key、聊天模型和 embedding 模型。API Key 加密保存，保存后不再返回明文。
- 工作区 owner 启用并维护该工作区的一份共享向量索引。文档 embedding 与重建费用由 owner 承担。
- 工作区固定共享索引所使用的 embedding 端点和模型。成员只有使用相同的端点和 embedding 模型，并配置自己的 API Key 后，才能检索该索引。
- 成员自行承担问题 embedding 和聊天生成的 API 费用。
- AI 支持三种模式：当前工作区、全部可访问工作区和聊天辅助。聊天辅助同时支持通用对话，以及当前文档或选中文本辅助。
- 本期对话历史只保存在浏览器会话中，不在服务端持久化。

## 架构

### 存储

PostgreSQL 镜像增加 `pgvector` 扩展。新增数据按以下所有权边界拆分：

1. 用户 AI 配置
   - 每位用户一份配置，包含 `baseUrl`、加密后的 API Key、聊天模型、embedding 模型和时间戳。
   - 使用仅服务端可用的密钥加密，独立于浏览器可见状态。日志和 API 响应只显示脱敏后的配置摘要。

2. 工作区 AI 设置
   - 每个工作区一份由 owner 管理的设置，记录是否启用索引、索引 owner、固定的 embedding 端点和模型，以及索引健康状态。
   - 设置中不保存 API Key 明文。修改模型或端点会使现有工作区索引失效，必须完整重建。

3. 已索引内容
   - 从持久化文档区块提取文本，切分为稳定片段，保存 `workspaceId`、`documentId`、`blockId`、来源顺序、内容哈希、向量和时间戳。
   - 工作区、文档和区块标识使删除和来源追溯可以精确执行。
   - 向量相似度索引按工作区及固定的 embedding 配置隔离。

4. 索引任务
   - 任务记录 `pending`、`processing`、`completed` 和 `failed` 状态、重试次数、来源版本，以及可安全展示的失败信息。
   - 按工作区和文档去重任务，连续快速编辑只保留一个最新任务，不会为每次输入都调用 API。

### 服务

服务端统一负责模型供应商集成，并提供边界明确的应用服务：

- AI 配置服务：校验 OpenAI 兼容配置、加密保存 Key，并返回脱敏状态。
- 工作区 AI 服务：允许 owner 启用索引、选择兼容的 embedding 配置、重建共享索引，并查看索引健康状态。
- 索引服务：提取并切分变化内容，跳过哈希未变化的片段，使用索引 owner 的凭据请求 embedding，并以事务方式写入或删除向量。
- 检索服务：每次检索前验证当前成员关系，使用提问者自己的 Key 对问题生成 embedding，按请求范围过滤向量，并返回来源元数据。
- 聊天服务：组合用户问题、可选的选中文本和检索片段，并流式返回 OpenAI 兼容聊天结果。

浏览器客户端不直接调用任何模型供应商。服务端也不信任客户端提供的工作区或文档成员关系声明。

## 数据流

### 索引

1. 服务端已持久化的文档通过正常工作区保存流程发生变化。
2. 应用将文档索引状态标记为待更新，并新建或更新该工作区-文档的索引任务。
3. 后台 worker 读取持久化文档快照，提取可索引文本区块并计算内容哈希。
4. 未变化的片段继续保留；新增或变化的片段使用工作区 owner 的 API Key 和该工作区固定的 embedding 配置生成向量。
5. worker 替换变化的向量，移除已删除来源区块的向量，并将任务标记为完成。

若 owner 移除了自己的配置、端点调用失败，或模型不再匹配，任务会以可恢复状态失败。编辑和协作功能仍正常工作。

### 提问

1. 客户端将问题和所选模式提交到服务端接口。
2. 服务端认证调用者，读取其加密配置；在需要知识库检索时，校验该配置是否与目标工作区的 embedding 配置一致。
3. 当前工作区模式只搜索当前工作区；全部工作区模式只搜索调用者当前仍是成员的工作区。
4. 服务端使用调用者的 Key 对问题生成 embedding，执行向量相似度搜索，并将最相关的来源片段打包为受约束上下文。
5. 聊天服务将问题和受约束上下文发送给调用者配置的聊天模型，并流式返回结果。
6. UI 显示回答以及其来源文档和区块链接。

通用聊天辅助不执行知识库检索。当前文档或选中文本辅助直接将该文本作为上下文，不要求工作区已建立向量索引。

## 界面

### AI 设置

每位用户都有 AI 设置页，用于填写 OpenAI 兼容的 `Base URL`、API Key、聊天模型和 embedding 模型。保存时执行连通性校验，之后只显示脱敏 Key 状态，UI 不再显示已保存 Key。

工作区 owner 额外拥有知识库控制项，可启用共享索引、设置固定的 embedding 配置、请求重建，并查看就绪、更新中、失败和不可用状态。非 owner 可以看到索引是否可用，但不能管理索引。

### AI 面板

编辑器提供带模式选择器的 AI 面板：

- 当前工作区：检索当前选中工作区中的文档。
- 全部工作区：检索调用者当前可以访问的全部工作区。
- 聊天辅助：在通用聊天与当前文档或选中文本辅助之间切换。

受知识库约束的模式展示可跳转到原始文档和区块的来源引用。回答流式显示在面板中。面板针对缺少用户配置、模型不兼容、索引不可用、没有相关来源、无权访问、供应商限流和供应商故障提供可操作状态。

## 权限、费用与隐私

- 每次检索均检查活跃工作区成员关系。不存在当前成员关系时，任何向量结果都不能跨工作区返回。
- 用户被移出工作区后，立即失去该工作区的 REST 和向量检索权限，保持现有生命周期行为一致。
- 索引 owner 的 Key 仅用于文档 embedding 与索引维护，绝不向成员暴露。
- 调用者自己的 Key 用于问题 embedding 与聊天生成。进行知识库检索时，调用者必须使用工作区固定的 embedding 端点和模型。
- 发送给模型供应商的内容仅包括问题、可选选中文本和回答所需的检索片段。API Key 不写入应用日志或客户端响应。

## 失败处理

- 索引任务对临时供应商故障使用有上限的指数退避重试；超过次数后保留失败状态，由 owner 手动重试。
- API 将无效凭据、模型不兼容、限流、请求超限和供应商故障映射为稳定的应用错误，不返回密钥或原始供应商响应。
- 索引缺失或过期时，受知识库约束的检索返回相应状态，但不会阻塞文档编辑。
- 内容哈希避免重复 embedding 调用。切换工作区 embedding 配置时，先使旧向量失效，再执行重建。
- 删除工作区、文档或区块时，同时删除其已索引片段和待处理任务。

## 测试

- 单元测试覆盖配置加密与脱敏、配置兼容性、文本分块、片段哈希、任务去重、上下文组装和检索范围过滤。
- PostgreSQL 集成测试运行在真实 `pgvector` 数据库中，验证迁移、向量写入、相似度排序、删除清理、任务重试和成员关系变化。
- API 测试模拟 OpenAI 兼容的 embedding 与流式聊天接口，验证请求构造、错误映射、Key 不泄露和来源归属。
- Playwright 测试覆盖 owner 建立索引、成员在三种模式下提问、来源跳转、模型不匹配状态，以及成员被移除后无法再检索。
- 当 AI 未配置或供应商不可用时，现有编辑器、协作和工作区生命周期测试仍必须通过。

## 不在本期范围内

- 语音转写和语音合成。
- 本地模型供应商和本地向量数据库。
- 已有服务端工作区内容以外的外部文件和文件夹导入。
- 持久化 AI 对话历史、AI 直接修改文档，以及超出调用者当前成员关系的跨工作区结果共享。
