# M8 编辑能力：多块操作与 Markdown 设计

状态：已确认

日期：2026-07-22

范围：M8.1B + M8.1C

## 1. 背景

M8.1A 为段落、标题、引用和待办建立结构化富文本与纯文本投影。当前编辑器已经支持单块原生拖拽、块层级、单块类型转换、附件、表格等复杂块，但缺少稳定的多块选择模型、可撤销的批量操作和跨文档内容交换。现有 Markdown 仅用于输入快捷转换，不提供文档级导入导出。

本阶段把 M8.1B 与 M8.1C 合并为“块操作与内容交换”能力：同一套规范化 Block 契约同时服务多块剪贴板、批量操作和 Markdown 转换，避免三套序列化规则相互偏离。

## 2. 目标

- 提供桌面、触摸和键盘均可用的 Notion 风格多块选择。
- 使用 `dnd-kit` 支持稳定的单块及多块拖拽、自动滚动和键盘排序。
- 批量执行复制、剪切、粘贴、复制块、删除、缩进、减少缩进、块类型转换和兼容的行内格式。
- 同工作区跨文档复制保持块结构、富文本、任务字段和附件；跨工作区安全降级不可移植数据。
- 使用 Unified/Remark AST 实现可定位错误的 Markdown 原子导入和确定性导出。
- 无附件时导出 `.md`，有附件时导出包含 `document.md` 与 `assets/` 的 ZIP。
- 所有批量变更都形成一个撤销单元，并与现有文档保存、历史和协同结构快照兼容。

## 3. 非目标

- 不支持矩形框选或画布式套索选择。
- 不支持通过拖拽直接改变块层级；层级变更使用缩进和减少缩进命令。
- 不支持批量添加链接、修改链接或插入 mention。
- 不把评论、协作光标、历史版本和审计记录复制到新块。
- 不提供 Markdown 源码编辑模式、实时双栏预览或任意 HTML 导入。
- 不承诺所有复杂块无损映射为标准 Markdown；无法表达的语义必须产生明确兼容性提示。
- 不建设 M9 的增量块 API、后台任务队列或通用导出任务系统。

## 4. 已确认决策

- 拖拽使用 `@dnd-kit/core`、`@dnd-kit/sortable` 和 `@dnd-kit/utilities`。
- 多选采用 `Shift+点击` 连续选择、`Ctrl/Cmd+点击` 增减单块；选择父块时自动包含其全部子孙块。
- 提供完整核心批量操作和跨块行内格式，批量链接与 mention 除外。
- 同工作区复制完整保真；跨工作区仅保留可移植内容，附件降级为文件名和安全链接。
- Markdown 使用 Unified、`remark-parse`、`remark-gfm` 与 `remark-stringify`。
- 导入采用原子模式；任何阻断错误都不得修改当前文档。
- 导入默认创建新文档，标题取首个顶层 H1，否则取文件名。
- 有附件的导出使用 ZIP 和相对资源路径。

## 5. 组件与模块边界

### 5.1 选择模型

新增独立的 Block 选择模块，职责只包括：

- 根据当前可见扁平顺序解析单选、范围选择和切换选择。
- 将显式选择展开为包含子孙块的有效选择。
- 计算操作根集合，移除“祖先与子孙同时作为移动根”造成的重复。
- 在远端删除、过滤变化或文档切换后修剪失效 ID。

建议契约：

```ts
interface BlockSelectionState {
  anchorBlockId: string | null;
  selectedBlockIds: string[];
}

interface ResolvedBlockSelection {
  orderedBlockIds: string[];
  rootBlockIds: string[];
}
```

选择状态是编辑器瞬态状态，不写入 Block、IndexedDB、PostgreSQL 或 Yjs。切换文档或按 `Escape` 时清空。进入块内文本编辑时清空多块选择，避免 DOM 文本选区与 Block 选区同时生效。登录 viewer 仍可选择并复制可读块，但工具条只显示复制；匿名分享页不启用 Block 多选。

### 5.2 批量操作模型

批量操作定义为纯文档变换：输入当前 `EditorDocument`、有序根 Block ID 和操作参数，输出新文档及一条可撤销记录。组件不得直接逐块调用现有单块回调，否则会产生多次保存和不完整撤销。

```ts
interface BatchBlockMutationResult {
  document: EditorDocument;
  focusBlockId: string | null;
  affectedBlockIds: string[];
}
```

批量删除、复制、移动和类型转换只刷新实际变化的块时间，并只刷新一次文档 `updatedAt`。操作找不到目标、会形成层级循环或无法保持所选根的相对顺序时返回原文档和稳定错误，不执行部分变更。

### 5.3 拖拽适配层

`dnd-kit` 只负责传感器、碰撞检测、拖拽覆盖层和自动滚动；文档顺序仍由纯模型函数计算。这样测试无需模拟浏览器拖拽坐标，协同层也只接收最终结构结果。

- Pointer Sensor 在移动 6 px 后激活，避免点击块控制误触拖拽。
- Touch Sensor 长按 180 ms 激活，并允许页面正常滚动。
- Keyboard Sensor 使用方向键移动，`Space` 拾取/放下，`Escape` 取消。
- 拖动已选块时移动整个操作根集合；拖动未选块时先把选择替换为该块。
- Drop 只提供同级 `before` / `after` 位置；缩进命令负责改变父级。
- 拖拽覆盖层显示首块摘要和总块数，不复制真实编辑器 DOM。
- 目标位于移动子树内部、目标已被删除或远端结构已使目标失效时取消操作。

## 6. 多块选择交互

### 6.1 桌面

- 点击块侧边控制区域选择单块。
- `Shift+点击` 从锚点到目标按当前可见顺序连续选择。
- `Ctrl/Cmd+点击` 增加或移除单块，并更新选择锚点。
- 父块加入选择后，其折叠或未折叠的全部子孙块都进入有效选择。
- 选择范围包含父子重复时，视觉上均标记，但移动与删除只使用操作根集合。
- 点击空白处、进入文本编辑、切换文档或按 `Escape` 清空选择。

### 6.2 移动端

- 长按块控制区域进入多选，后续点按切换单块。
- 进入多选后显示固定底部批量工具条，避免依赖 hover。
- 工具条和拖拽覆盖层必须限制在视口内，不遮挡系统安全区域。

### 6.3 视觉与可访问性

- 选择背景使用项目灰白色调、浅灰边框和左侧强调线，不使用纯黑或深灰面板。
- 批量工具条使用 Lucide 图标、固定尺寸、tooltip、`aria-label` 和 `aria-pressed`。
- 每个选中 Block 使用 `aria-selected="true"`；工具条状态区域通过 `aria-live="polite"` 报告选择数量和操作结果。
- 键盘焦点与视觉选择分别管理，不能因远端更新无提示地跳到其他块。

## 7. 批量操作语义

### 7.1 复制、剪切、粘贴和复制块

- 复制按文档顺序序列化所选根及其子孙块。
- 所有新块生成新 ID，`parentId` 和 `children` 只重映射包内关系。
- 评论、评论解决状态、协作 presence 和历史身份永不复制。
- 同工作区保留富文本、块类型、checked、status、dueDate 和仍有效的 assignee。
- 跨工作区清空 assignee；附件、内部对象 key 和不可访问的文档目标不进入结构化载荷。
- 剪切只有在粘贴成功后删除源块；失败时源文档保持不变。
- 复制块等价于“内部复制后紧邻原选择粘贴”，并形成一个撤销单元。

### 7.2 删除、移动和层级

- 删除操作根会同时删除其子树；删除后至少保留一个空段落。
- 缩进要求每个操作根存在可用的前一个同级块，且不会形成循环。
- 减少缩进把操作根提升一级，并保持根集合和子树的相对顺序。
- 一次多块拖拽只产生一个结构快照和一个撤销记录。

### 7.3 类型转换和批量格式

- 类型转换只接受现有命令目录支持的目标类型。
- 多个文本块可统一转换为段落、标题、引用、待办或代码；转换到复杂块需要单块专用参数，因此不作为批量目标。
- 粗体、斜体、删除线和行内代码对每个兼容文本块的完整正文生效。
- 混合状态遵循“不是全部激活时统一添加，全部激活时统一移除”。
- 代码和复杂块在批量 marks 操作中保持不变；工具条报告实际处理数量。
- 链接需要单一 URL 和精确文本范围，mention 需要单一目标，因此两者不提供批量操作。

## 8. 内部剪贴板契约

浏览器同时写入：

- `application/x-nexus-blocks+json`：Nexus 结构化载荷。
- `text/html`：供其他富文本应用使用的安全 HTML。
- `text/plain`：按块顺序生成的纯文本和换行。

结构化载荷必须包含：

```ts
interface NexusBlockClipboardPayload {
  version: 1;
  sourceWorkspaceId: string;
  sourceDocumentId: string;
  copiedAt: number;
  blocks: ClipboardBlockSnapshot[];
}

interface ClipboardBlockSnapshot {
  sourceId: string;
  sourceParentId: string | null;
  sourceChildren: string[];
  type: BlockType;
  headingLevel: HeadingLevel;
  content: string;
  richText: RichTextDocument | null;
  data: BlockData | null;
  checked: boolean;
  assignee: string;
  dueDate: string;
  status: BlockStatus;
}
```

快照类型显式排除 `comments`、`createdAt` 和 `updatedAt`；粘贴端重新生成身份与时间。附件对象 key 只存在自定义 MIME 中且仍需服务端按来源工作区复验，HTML/纯文本格式不得携带 key。

读取时执行完整白名单校验、256 KB 单富文本限制和 2 MB 总载荷限制。未知版本、未知 Block 类型或非法关系直接忽略结构化 MIME，回退到安全 HTML/纯文本粘贴。不得信任剪贴板中的工作区 ID、对象 key、用户 ID 或 URL。

### 8.1 附件复制

同工作区目标文档粘贴附件时，通过受授权的附件复制 API 为每个附件生成新对象 key 和数据库记录。对象复制全部成功后才提交文档内容；对象复制或后续文档保存任一步失败时清理本次创建的对象并保持目标文档不变。

跨工作区不复制对象：图片和文件块转换为普通段落，正文包含原文件名；仅当原 URL 通过安全链接校验且对目标用户可公开访问时附加链接，否则只保留文件名。

## 9. 协同、保存与撤销

- 多块操作基于执行瞬间的最新文档结构解析 ID，不基于拖拽开始时保存的数组索引。
- Yjs 文本 Fragment 继续负责块内正文；批量结构操作通过现有文档结构记录同步。
- 批量 marks 逐块写入对应 XmlFragment，但 UI 将整次命令记录为一个逻辑事务来源。
- 延迟父快照不得覆盖已合并的 Fragment 内容。
- 远端删除选中块时自动修剪选择；远端移动不会改变剩余选择的相对顺序定义。
- 撤销只对当前用户发起的最后一个本地批量命令生效；远端并发操作不进入本地撤销栈。
- 粘贴、批量删除、批量格式和多块移动各自只触发一次顶层保存调度。

## 10. Markdown 转换架构

### 10.1 模块边界

共享 Markdown 模块只处理纯数据，不依赖 React、DOM 或文件下载。解析使用 Unified/Remark，ZIP 编解码使用固定版本的 `fflate`：

- `parseMarkdownDocument(source, options)`：Markdown 到规范化文档草稿和诊断。
- `serializeDocumentToMarkdown(document, options)`：文档到 Markdown 和资源清单。
- `validateMarkdownArchive(entries)`：ZIP 路径、资源大小和清单一致性。
- `resolveMarkdownImportTitle(ast, filename)`：首个顶层 H1 或文件名。

UI 层负责选择文件、展示预览、确认、下载和错误状态；服务层负责附件暂存、新文档创建和补偿清理。

### 10.2 支持映射

| Markdown | Nexus |
| --- | --- |
| 段落 | paragraph Block |
| H1-H6 | heading Block + headingLevel |
| `>` 引用 | quote Block |
| fenced code | code Block |
| `- [ ]` / `- [x]` | todo Block |
| 无序/有序列表 | bulletedList / numberedList Block |
| 嵌套列表 | 使用 parentId/children，最大 10 层 |
| `---` | divider Block |
| GFM 表格 | table Block |
| `**`、`*`、`~~`、反引号 | M8.1A marks |
| 标准链接 | M8.1A 安全 link mark |
| 硬换行 | hardBreak |
| `math` fenced code | formula Block |

首个顶层 H1 用作新文档标题且不重复生成正文 Block；后续 H1 正常生成 heading Block。没有正文时创建一个空段落。

### 10.3 不可直接表达的结构

- 人员、任务和日期 mention 导出为 `@label` 普通文本，不尝试在重新导入时恢复目标 ID。
- 文档 mention 导出为安全的 `/documents/{publicId}` 链接，重新导入后保持链接而非自动恢复 mention。
- linkCard 导出为普通链接。
- toggle 导出摘要和展开后的子块，丢失折叠状态并产生警告。
- kanban 导出为列标题与卡片列表，丢失看板身份并产生警告。
- 附件引用 `assets/{稳定文件名}`；重名文件追加确定性的短后缀。
- 原始 HTML、脚本、iframe、未知指令和无法映射的自定义节点是阻断错误。

导出不能静默丢弃块。所有降级项都进入导出兼容性报告；报告不写入 Markdown 正文。

## 11. 导入流程

### 11.1 文件与预览

支持单个 UTF-8 `.md` 和 Nexus 导出的 `.zip`。流程为：

1. 在浏览器读取文件头并验证扩展名、压缩条目数量和声明大小。
2. 使用 Remark 生成带源码位置的 AST。
3. 把 AST 转换为规范化文档草稿和诊断列表。
4. 显示标题、块数、附件数、警告和阻断错误。
5. 只有零阻断错误时才允许确认。
6. 确认时把原始文件及 SHA-256 发送给服务端；服务端使用同一共享模块重新解析和校验后创建新文档，不覆盖当前文档。

诊断包含 `severity`、`line`、`column`、`code` 和中文 `message`。语法可解析但发生显式安全降级时使用 warning；未知结构、非法链接、超限和损坏资源使用 error。

### 11.2 原子创建与附件

- 普通 `.md` 中的远程图片不由服务端主动抓取，避免 SSRF；它们降级为带 alt 的安全链接并显示警告。
- 浏览器预览是交互反馈，不是安全边界；服务端必须重新解析原始 Markdown/ZIP，且客户端摘要哈希与上传文件不一致时拒绝。
- ZIP 资源先写入工作区临时前缀，校验 MIME、大小和哈希后再创建文档。
- 数据库事务创建文档、Block、附件记录和对象引用。
- 数据库失败时删除临时前缀；对象提升失败时回滚数据库并清理已提升对象。
- 成功后返回新文档 public ID，并导航到新文档。

### 11.3 限制

- Markdown 源文件最大 2 MB。
- 转换后最多 5000 个 Block，单文档层级最大 10。
- ZIP 最多 200 个条目，解压后总大小最大 100 MB，单附件继续遵守现有上传限制。
- ZIP 只接受相对路径；拒绝绝对路径、`..`、反斜杠逃逸、重复规范化路径和符号链接。
- UTF-8 解码错误、加密 ZIP 和压缩炸弹特征直接拒绝。

## 12. 导出流程

- 具有文档读取权限的登录用户可以导出；匿名分享页不提供导出入口。
- 导出基于服务端重新读取的授权快照，不接受客户端提交文档内容。
- 无附件时响应 `text/markdown`，文件名使用清洗后的文档标题。
- 有附件时响应 ZIP，根目录包含 `document.md`、`assets/` 和机器可读的 `manifest.json`。
- `manifest.json` 只记录格式版本、资源相对路径、大小和 SHA-256，不包含用户、工作区或权限信息。
- 对象读取失败时整个导出失败并列出缺失资源，不生成残缺 ZIP。
- 导出响应使用流式传输；第一版不落盘保存导出包。

本地 IndexedDB 模式支持无附件 `.md` 的导入导出和所有多块操作。由于本地模式没有受授权的对象存储，包含附件的 ZIP 导入入口隐藏；已有本地附件只按安全可访问链接导出并显示非便携警告。远程 PostgreSQL 模式提供完整 ZIP 资源往返。

## 13. 权限与安全

- 多选编辑、剪切、粘贴、导入和批量操作要求文档写权限。
- 复制和导出要求读权限；剪贴板不得包含未展示的评论、审计或成员数据。
- 同工作区附件复制同时验证源文档读权限、目标文档写权限和对象归属。
- Markdown 链接复用 M8.1A 安全协议规则；导入不执行 HTML、JavaScript 或数据 URL。
- ZIP 文件名只作为显示建议，不作为未经清洗的文件系统路径。
- 服务端重新校验结构化剪贴板和 Markdown 转换结果，不能依赖浏览器校验。

## 14. 错误处理

- 选择中的块被远端删除时修剪选择并显示非阻断状态，不操作其他块。
- 批量操作前置条件失败时不产生部分文档、不写历史、不改变撤销栈。
- 剪贴板权限被浏览器拒绝时提供系统剪贴板失败状态，不伪装成功。
- 导入错误保留文件选择和诊断列表，允许用户修复原文件后重新选择。
- 附件复制或 ZIP 创建失败时执行补偿清理并返回稳定错误码。
- 下载被用户取消不记录为服务端失败。

## 15. 迁移与回滚

- 多块选择和剪贴板不需要数据库迁移。
- 引入 `dnd-kit`、Unified、Remark、GFM 和 ZIP 流式库时固定版本并记录许可证。
- 内部剪贴板从 `version: 1` 开始；未来未知版本安全回退到 HTML/纯文本。
- 新导入功能只创建标准现有文档和 Block，回滚应用不会产生旧服务无法读取的数据。
- 附件复制沿用现有对象和附件表；不为跨工作区复制扩大对象访问边界。
- 回滚前端后，已导入文档仍可正常编辑；只失去批量 UI 和导入导出入口。

## 16. 测试策略

- 选择模型单元测试覆盖范围选择、切换、父子展开、操作根去重和远端删除修剪。
- 批量模型测试覆盖顺序、循环保护、缩进、删除最后块、mixed marks 和单次时间刷新。
- `dnd-kit` 组件测试覆盖指针、触摸、键盘、自动滚动、取消和只读隐藏。
- 剪贴板测试覆盖 MIME 优先级、ID 重映射、关系校验、未知版本和跨工作区降级。
- 附件集成测试覆盖对象复制、权限、事务失败和补偿清理。
- Markdown 黄金文件测试覆盖所有支持结构、中文内容、确定性导出和导入再导出。
- 诊断测试覆盖准确行列、非法链接、原始 HTML、超限、损坏 ZIP 和路径穿越。
- 协同测试覆盖多块移动、批量 marks、远端删除和一次逻辑撤销。
- Playwright 覆盖桌面/移动多选、拖拽、批量工具条、导入预览、错误列表和下载包内容。
- 500 个普通文本块下测量选择、拖拽和批量操作，禁止明显布局跳动。

## 17. 验收标准

- 桌面、移动和键盘用户都能选择并移动多个块，选择父块不会丢失子树。
- 所有批量操作保持块顺序和层级，并作为一个撤销单元。
- 同工作区跨文档复制完整保留支持的数据；跨工作区不泄露对象 key、成员或私有 URL。
- 批量格式只作用于兼容文本块，混合状态和处理数量表达准确。
- Markdown 支持表内列出的结构，导入错误能准确定位到源码行列。
- 原子导入失败后当前文档、目标文档目录和对象存储均无半成品。
- 无附件导出为可读 `.md`，有附件导出包可离线打开且资源路径有效。
- 协同、刷新、历史和匿名权限边界不因批量操作或导入导出发生回归。
- 灰白批量工具条、拖拽覆盖层和导入预览在桌面与移动视口内无溢出或重叠。
