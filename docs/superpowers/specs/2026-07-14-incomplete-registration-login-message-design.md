# 未完成注册登录提示设计

## 背景

密码登录会明确区分邮箱不存在、未设置密码、邮箱未验证和密码错误。当前已提交注册信息但尚未通过邮箱验证码的账号会返回“邮箱尚未验证，请先输入邮件中的验证码”，但登录页没有验证码输入框，提示无法引导用户完成下一步操作。

## 目标

当密码登录命中 `email_not_verified` 状态时，显示：

> 该邮箱尚未完成注册，请先完成注册再登录

该文案说明账号仍处于未完成注册状态，并引导用户回到注册流程。

## 设计

- 保留稳定错误代码 `email_not_verified`，不把该状态伪装成邮箱不存在。
- 保留 HTTP `403` 状态，不改变 API 响应结构。
- 只修改 `email_not_verified` 的用户可见中文消息。
- 登录页继续显示服务端返回的业务错误，不增加前端特殊映射或自动页面切换。
- 注册、验证码重发、邮箱验证和密码找回流程保持不变。

## 数据流

1. 用户提交邮箱和密码登录。
2. 存储层查到账号已有密码但 `email_verified_at` 为空。
3. 存储层抛出代码为 `email_not_verified` 的领域错误，并携带新文案。
4. API 继续将该错误映射为 `403` JSON 响应。
5. 登录页在现有错误提示区显示新文案。

## 测试

- 新注册账号尚未验证时，密码登录返回新文案。
- 无密码遗留账号补齐注册密码但尚未验证时，密码登录返回新文案。
- 完全不存在的邮箱继续返回“该邮箱尚未注册，请先创建账号”。
- 邮箱验证完成后，正确密码仍可正常登录。

## 非目标

- 不自动切换到注册页或验证码输入页。
- 不改变错误代码、HTTP 状态或数据库账号状态。
- 不修改其他认证提示文案。
