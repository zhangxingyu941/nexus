# 验证码重发冷却与邮件样式设计

## 目标

保持邮箱验证和密码重置验证码 10 分钟有效，为同一账号增加 60 秒重发冷却，在认证界面展示倒计时，并发送带纯文本降级版本的灰白简约 HTML 邮件。

## 当前状态

- 验证码固定为 6 位数字，数据库仅保存按用户和用途限定的 HMAC。
- `auth_tokens.expires_at` 已实现 10 分钟有效期。
- 当前发送新验证码时会立即删除旧验证码，但没有重发冷却。
- 注册验证和密码重置共用 `auth_tokens`，用途分别为 `verify-email` 和 `reset-password`。
- 邮件当前只发送纯文本正文。

## 冷却机制

PostgreSQL 是重发冷却的权威数据源。`auth_tokens.created_at` 已记录最近发送时间，因此不新增数据表或 Redis 键。

为已有账号创建验证码时，在事务中使用 `FOR UPDATE` 锁定对应的 `app_users` 行。持有锁后，认证存储读取该用途当前验证码的创建时间。如果 `created_at + 60 秒` 晚于当前时间，则抛出带向上取整剩余秒数的类型化冷却错误；否则删除旧验证码，并插入 HMAC 和 `now + 10 分钟` 的过期时间。

账号行锁会串行化并发重发请求，避免两个请求同时通过冷却检查并发送两封邮件。首次注册仍使用邮箱唯一约束作为并发冲突的最终保护；账号创建后，所有重发都进入加锁路径。

冷却按验证码用途分别计算。注册验证只面向未验证账号，密码重置只对已存在账号有效，两个流程在正常业务中不会互相干扰。

## API 行为

### 注册

- 首次成功保持 `201` 状态，响应体增加为 `{ registered: true, retryAfterSeconds: 60 }`。
- 冷却期内重发返回 `429`，同时返回 `Retry-After: <seconds>` 和 `{ error, retryAfterSeconds }`。
- 响应中永远不包含验证码。

### 找回密码

- 邮箱格式有效时始终返回 `202 { accepted: true, retryAfterSeconds: 60 }`。
- 邮箱不存在和账号处于冷却期时，响应状态与内容完全相同。
- 冷却期内不创建新验证码，也不发送邮件。
- 该行为在为前端提供稳定倒计时的同时，继续防止通过接口枚举账号。

现有 Redis 每邮箱和每 IP 的小时级限流继续保留。PostgreSQL 的 60 秒冷却是额外的账号级业务约束，不替代滥用防护。

## 前端行为

首次注册或找回密码请求成功后，对应验证码输入界面启动 60 秒截止时间。重发按钮在冷却期间禁用，并显示 `重新发送（59s）`，每秒递减；归零后恢复为 `重新发送验证码`。

注册接口返回 `429` 时，前端读取 `retryAfterSeconds`，按服务端剩余时间重新启动倒计时并显示错误。前端计时器只负责反馈，不承担安全校验；刷新页面或直接调用 API 仍会受到数据库冷却限制。

计时器只属于当前认证流程，在组件卸载或离开验证码输入模式时清理。

## 邮件样式

每封认证邮件同时包含 `text` 和 `html` 正文。HTML 使用表格布局和内联 CSS，以兼容 QQ 邮箱及常见桌面、移动邮件客户端。

视觉规范：

- 页面背景为浅灰色，正文容器为白色。
- 最大宽度 560px，使用 `#e4e4e7` 浅灰边框和 8px 圆角。
- 顶部显示紧凑的 Nexus 品牌名和对应流程标题。
- 6 位验证码置于高对比度浅灰区域，并使用充足的字符间距。
- 明确展示“10 分钟内有效”和对应流程的安全提示。
- 不加载远程图片、网络字体、渐变、跟踪像素、脚本或交互控件。

用户显示名称在插入 HTML 前必须转义。验证码只允许进入邮件正文和显式 E2E 捕获文件，不进入邮件主题、应用日志或 API 响应。

## 错误处理

- 使用类型化冷却错误区分预期重发限制与数据库、SMTP 故障。
- 注册接口把冷却错误映射为 `429`，其他校验错误保持现有稳定响应。
- 找回密码接口隐藏冷却差异，继续返回不可枚举的 `202` 响应。
- SMTP 失败保持现有行为：注册流程提示邮件暂时无法发送；找回密码流程不披露账号是否存在。

## 测试范围

- 存储测试覆盖发送后第 0 秒和第 59 秒拒绝、第 60 秒允许、10 分钟过期、仅最新验证码有效、两种用途和并发请求串行化。
- 路由测试覆盖注册 `429` 与 `Retry-After`、找回密码响应一致性，以及冷却期间不调用邮件发送器。
- UI 测试使用可控计时器验证按钮禁用、倒计时文本、60 秒后恢复，以及服务端剩余时间纠正。
- 邮件测试验证纯文本和 HTML 正文、显示名称转义、验证码突出显示、10 分钟提示、用途提示，以及不包含验证或重置链接。
- 源码修改完成后运行聚焦测试、完整 Vitest、TypeScript 和一次生产构建。

## 不包含范围

- 不修改验证码长度或 10 分钟有效期。
- 不在浏览器刷新后持久化前端倒计时。
- 不引入外部邮件模板框架或远程视觉资源。
- 不替换现有 Redis 小时级滥用限流。
