# 协同基线设计

## 目标

推进当前项目的 1/2/3 阶段：先固化 Next 结构重构基线，再增加协同前置数据层，最后接入最小 WebSocket/Yjs 协同雏形。

本阶段不提交 git commit。

## 范围

### 阶段 1：基线稳定

- 运行完整测试和 Next build。
- 更新当前开发进度文档。
- 保留当前未提交改动，不创建提交。

### 阶段 2：同步事件层

- 为工作区/文档/块操作增加事件描述能力。
- 事件层不替代现有纯函数，而是描述状态变化，供后续协同层消费。
- 继续保留当前整份工作区保存逻辑和 IndexedDB 兜底。

### 阶段 3：最小协同

- 引入 Yjs 作为协同数据结构。
- 引入本地 WebSocket 协同服务，支持多个浏览器窗口加入同一文档房间。
- 最小同步范围是当前文档的块内容。
- UI 增加协同连接状态，但不做登录、权限、数据库、历史版本或复杂光标。

## 架构

- `src/features/editor/model/workspaceEvents.ts` 定义领域事件、事件创建工具和事件应用/描述函数。
- `src/features/editor/collaboration/` 放浏览器端协同状态、Yjs 映射和连接工具。
- `server/` 保留开发专用 WebSocket 协同服务脚本。
- `EditorPage` 继续拥有工作区状态，并通过协同 hook 接收远端块内容更新。

## 数据流

1. 本地编辑仍先更新 React 工作区。
2. 保存逻辑继续调用 `saveSyncedWorkspace`。
3. 协同开启时，块内容变更同步到 Yjs document。
4. 其他窗口收到 Yjs 更新后，把远端块内容补丁应用回当前工作区。
5. 如果协同服务不可用，UI 显示离线，本地编辑和保存继续工作。

## 验证

- `pnpm test -- --run`
- `pnpm build`
- 针对事件层增加模型单元测试。
- 针对协同映射增加纯函数测试。
