# Nexus 功能与使用路径

本文说明当前已交付的主要能力，以及它们在界面中的入口。需要本地启动方式时，请参阅[本地开发](development.md)；生产环境请参阅[生产部署](deployment.md)。

## 编辑与内容

Nexus 以区块组织内容，当前支持段落、标题、待办、引用、代码、图片、文件、表格和看板。段落、标题、引用和待办使用结构化 TipTap/ProseMirror JSON 保存，同时保留纯文本投影以支持搜索、历史和兼容读取。

编辑器支持粗体、斜体、删除线、行内代码、安全链接、mention、块内换行、Slash 菜单、评论、区块拖拽和长文档虚拟列表。浏览器本地模式使用 IndexedDB 作为离线兜底；数据库模式则由 PostgreSQL 保存工作区与文档内容。

## 协同

同一工作区成员可通过 Yjs WebSocket 实时同步正文、结构和在线状态。协同数据使用 PostgreSQL 快照与增量更新持久化，Redis 用于多实例间的实时传播。

协作房间始终绑定工作区和文档：`workspace:{workspaceId}:document:{documentId}`。成员资格或文档访问权限被收回后，已有 WebSocket 连接会被关闭。

## 工作区与角色

| 角色 | 能力 |
|---|---|
| owner | 管理工作区、邀请成员、调整角色、移除成员、转让所有权、删除或恢复工作区、管理分享链接 |
| editor | 编辑有写权限的文档，不能管理成员或工作区设置 |
| viewer | 读取被授权内容，不能写入或加入可写协作通道 |

工作区支持创建、搜索、切换、owner 重命名、成员管理、软删除和 7 天回收站恢复。最后一名 owner 不能降级、移除或退出工作区。

## 邀请成员

邀请功能只在数据库工作区模式下提供，且只有当前工作区的 owner 能发送邀请。

1. 在左侧边栏点击当前工作区名称。
2. 在工作区列表中点击该工作区的“管理”。
3. 打开“邀请”标签页，填写成员邮箱并选择 `editor` 或 `viewer`。
4. 发送后可在同一标签页重发或撤销邀请。

邀请默认 24 小时有效。收件人可通过邮件链接或应用内邀请中心接受、拒绝邀请。SMTP 投递失败不会丢失已创建的邀请，但会返回投递警告。

直接文档页面和“成员与协作”抽屉不提供发送邀请入口：前者聚焦单篇文档，后者只显示成员与在线状态。

## 文档权限与分享

文档支持三种访问策略：

- 私有：仅作者、显式授权成员和工作区 owner 可访问。
- 团队：继承工作区成员权限。
- 链接：通过匿名只读分享链接访问。

owner 可创建、重新生成或关闭匿名链接。链接默认有效期为 24 小时，可设置为 1 小时、7 天、30 天或最长 365 天。令牌仅在创建或重新生成时返回一次，数据库只保存 HMAC。匿名页面不会加载工作区导航、成员、评论、历史、任务元数据或协同服务。

## 认证与安全

Nexus 支持邮箱密码注册、6 位邮箱验证码、密码重置和可选 GitHub OAuth。敏感凭据在浏览器中通过 JWE 加密，服务端使用 Argon2id 存储密码哈希，并以 HttpOnly、SameSite Cookie 维护会话。

注册、登录、重置和 OAuth 使用 Redis 限流；认证与工作区操作写入不包含密码、验证码或原始令牌的审计记录。

## 多块操作与 Markdown

编辑器支持连续和增减的多块选择、批量复制/剪切/粘贴/删除、缩进、类型转换和跨块行内格式。多块拖拽支持鼠标、触摸和键盘操作；同工作区的结构化复制保留富文本与可访问附件，跨工作区会安全降级不可移植数据。

登录用户可从文档顶栏导出 Markdown；有写权限的用户还可导入 `.md` 创建新文档。无附件文档导出为 Markdown，有附件时远程工作区会生成含 `document.md`、`assets/` 与校验清单的 ZIP。导入在浏览器预览后仍由服务端重新解析和校验，损坏、超限或不安全的归档不会创建部分文档。
