# M8 开发状态

更新时间：2026-07-23

总体状态：M8.1A 结构化富文本、M8.1B 多块操作和 M8.1C Markdown 文档交换均已完成实现并纳入本地自动化验证。真实 PostgreSQL、完整协同和服务级端到端发布验收仍待具备 Docker/测试数据库环境后执行。M8.2 和 M8.3 尚未开始。

## M8.1A 结构化富文本

状态：功能实现完成，发布验收待补充

已完成：

- 文本类 Block 使用 `richText JSON + content 纯文本投影` 双字段；段落、标题、引用和待办使用规范化的单段 TipTap/ProseMirror JSON，代码和复杂 Block 固定为 `null`。
- 共享编解码模块白名单校验 marks、mention、hardBreak 和安全链接，限制单块 JSON 为 256 KB；服务端始终从 JSON 重算纯文本投影。
- PostgreSQL 新增可空 `editor_blocks.rich_text JSONB`，保留旧 `content`，以惰性读取和首次合法写入升级旧数据；IndexedDB v2 无需升级 object store。
- 文档、工作区、历史、远程补丁和 Yjs 块内容记录显式携带富文本；协同 fragment 首次按结构化正文种子化，之后不被延迟父快照覆盖。
- 选区工具条支持粗体、斜体、删除线、行内代码、链接和评论；链接使用锚定浮层，拒绝危险协议并支持键盘提交、关闭、打开、复制和移除。
- 外部 HTML 粘贴仅保留受支持行内格式和安全链接，多段内容转为块内换行；Nexus 内部剪贴板可保留完整 mention。
- 匿名快照保留公开格式与安全链接，同时把 mention 降级为普通文本，移除目标 ID 和类型。

## M8.1B 多块操作与拖拽增强

状态：功能实现完成，本地自动化验证完成

已完成：

- 提供连续、增减和父子孙展开的多块选择模型；在文档切换、文本编辑和远端删除后自动清理失效选择。
- 批量复制、剪切、粘贴、删除、复制块、缩进、减少缩进、类型转换与跨块行内格式全部通过纯文档变换执行，并作为单个撤销单元保存。
- 自定义剪贴板载荷使用白名单校验和尺寸限制；同工作区保留结构化富文本、任务字段与可访问附件，跨工作区安全降级不可移植数据。
- 使用 `dnd-kit` 提供鼠标、触摸和键盘可访问的多块排序与自动滚动；拖拽只计算最终结构变换，不直接操作编辑器 DOM。
- 桌面工具条与移动端固定批量工具条均提供图标、提示、ARIA 状态和操作反馈。

## M8.1C Markdown 文档交换

状态：功能实现完成，本地自动化验证完成

已完成：

- 基于 Unified/Remark 与 GFM 实现 Markdown 到规范化 Block 文档的解析，覆盖文本、标题、引用、代码、任务、嵌套列表、分割线、表格、公式、行内格式、安全链接和硬换行。
- 提供确定性 Markdown 序列化；对 mention、toggle、看板、链接卡片和不可直接表达的内容生成明确的兼容性诊断，不静默丢弃内容。
- 无附件文档导出为 `.md`；含可移植附件的文档导出为包含 `document.md`、`assets/` 与 SHA-256 `manifest.json` 的 ZIP。
- ZIP 导入校验路径遍历、重复路径、加密条目、UTF-8、条目数、压缩后总大小、单文件大小与资源哈希；导入从服务端重新解析原始字节，不信任客户端预览。
- 导入服务先落地资源、再创建文档，并在失败时执行对象和附件记录的补偿清理；导出始终从已授权的服务端文档快照生成。
- 编辑器顶栏为有读取权限的用户提供 Markdown 导出，为有写入权限的用户提供 Markdown 导入；本地模式支持无附件 Markdown 导入导出，远程模式支持完整附件 ZIP 往返。
