菜单项、悬浮提示和快捷键中心都读取同一份快捷键定义；系统根据 Windows/Linux 与 macOS 自动显示 Ctrl 或 ⌘。选中文字时 Ctrl + K 添加链接，未选中文字时打开全局搜索。

01 / 自身风格蓝色 Focus Rail

蓝色只表达当前操作焦点和任务进度；它连接块手柄、操作条和右侧上下文，形成 Nexus 的识别点。

02 / 直接编辑正文不是输入框

点击文字直接定位光标，聚焦仅显示 Focus Rail；不出现输入边框、底色或表单式文本框。

03 / Markdown快捷转换，不是源码模式

触发符完成后立即转换为富文本块；标题类型直接显示为 H1 到 H6。

04 / 插入分类Slash 锚定光标出现

不使用弹窗和遮罩；最近使用、标题、列表任务、媒体和数据内容分组展示，并支持别名搜索。

05 / 统一引用@ 使用行内选择层

People、Docs、Tasks 和 Dates 跟随光标出现；插入后成为可点击、可同步的行内实体。

06 / 快捷键一致性单一快捷键注册表

行为、菜单标签和快捷键中心共用定义，并按作用域处理冲突；第一版不开放自定义。

07 / 协作光标仅远程光标显示人员身份

本地光标保持 TipTap 默认样式且不显示名称；远程光标、姓名标签和选区共享人员色。标签靠近视口边缘时自动翻转，断线或离开文档后立即清除。

08 / 连续聚焦完成选择后立即进入下一步输入

打开 Popover 前保存当前选区；Text、H1-H6、Todo、Quote 和 Code 选择后直接聚焦正文，Table 聚焦首个单元格，Image/File 聚焦上传或说明区域，Board 聚焦首个卡片标题，Mention 插入后把光标放在引用后方。Esc 关闭时恢复原光标，不产生额外点击。
